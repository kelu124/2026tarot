# Random DataFrame Sampler for GitHub Pages

This project creates an interactive webpage that randomly selects 3 items from each of 4 pandas dataframes and displays them.

## üìÅ Files Included

1. **index.html** - Standalone version with data embedded in the HTML (best for small datasets)
2. **index_external_data.html** - Version that loads data from external JSON file (best for larger datasets)
3. **convert_dataframes.py** - Python script to convert your pandas dataframes to JSON
4. **data.json** - Example JSON data file (generated by the Python script)

## üöÄ Quick Start

### Option 1: Embedded Data (Simplest)

1. Edit the `data` object in `index.html` with your actual data
2. Upload `index.html` to your GitHub repository
3. Enable GitHub Pages in repository settings
4. Done! Your page is live

### Option 2: External JSON Data (Recommended for larger datasets)

1. **Prepare your data:**
   ```python
   # Install pandas if needed
   pip install pandas
   
   # Run the conversion script with your dataframes
   python convert_dataframes.py
   ```

2. **Customize the HTML:**
   - Open `index_external_data.html`
   - Update the `columns` arrays in the `dataframes` variable to match your data columns
   - Rename it to `index.html`

3. **Upload to GitHub:**
   ```bash
   # Create a new repository on GitHub, then:
   git init
   git add index.html data.json
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin https://github.com/yourusername/your-repo-name.git
   git push -u origin main
   ```

4. **Enable GitHub Pages:**
   - Go to your repository on GitHub
   - Click "Settings" ‚Üí "Pages"
   - Under "Source", select "main" branch
   - Click "Save"
   - Your site will be live at `https://yourusername.github.io/your-repo-name/`

## üìä Data Preparation Tips

### Keep JSON Files Small
- GitHub Pages works best with files under 100MB
- If your data is large, consider filtering to only essential columns
- You can also sample your data before converting to JSON

```python
# Example: Take only a subset of columns and rows
df1_filtered = df1[['col1', 'col2', 'col3']].head(1000)
```

### Handle Different Data Types

```python
# Convert dates to strings
df['date_column'] = df['date_column'].astype(str)

# Round decimals
df['float_column'] = df['float_column'].round(2)

# Handle NaN values
df = df.fillna('')  # or df.dropna()
```

### Unicode and Special Characters

```python
# Ensure proper encoding
with open('data.json', 'w', encoding='utf-8') as f:
    json.dump(data, f, ensure_ascii=False, indent=2)
```

## üîÑ How It Works

1. **On Page Load**: Automatically displays 3 random items from each dataframe
2. **Button Click**: Generates a new random selection
3. **Page Refresh**: Generates a new random selection

The randomization happens client-side using JavaScript's `Math.random()`.

## üêõ Troubleshooting

### "Error Loading Data" message
- Make sure `data.json` is in the same directory as `index.html`
- Check that the JSON file is valid (use a JSON validator)
- Ensure GitHub Pages is serving the files (check the repository settings)

### Columns not displaying
- Verify that the column names in the HTML match exactly with your JSON keys
- Check the browser console (F12) for error messages

### Data not updating
- Hard refresh the page (Ctrl+Shift+R or Cmd+Shift+R)
- Clear browser cache

## üìù Notes

- GitHub Pages only serves static files (HTML, CSS, JavaScript, JSON)
- All processing happens in the user's browser
- No server-side code can run on GitHub Pages
- Data is public - don't include sensitive information
